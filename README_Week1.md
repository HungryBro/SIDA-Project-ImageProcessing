## 📖 ทฤษฎีเบื้องหลัง (Theoretical Background)

### 1. Class Activation Mapping (CAM) คืออะไร?
Class Activation Mapping คือเทคนิคการสร้างแผนภาพความร้อน (Heatmap) ทับซ้อนบนรูปภาพต้นฉบับ เพื่อแสดงให้เห็นว่าพิกเซลหรือพื้นที่ส่วนใดในรูปภาพส่งผลต่อการตัดสินใจของโมเดลในการจำแนกหรือระบุตำแหน่งของวัตถุมากที่สุด (ตัวอย่างเช่น การวิเคราะห์ว่าโมเดลโฟกัสที่จุดใดในการแยกแยะสุนัขหรือแมว) โดยสีโทนร้อน (เช่น สีแดง, สีส้ม) จะแสดงถึงบริเวณที่มีระดับความสำคัญสูง

### 2. เทคนิค CAM รูปแบบต่างๆ และข้อดี/ข้อเสียเปรียบเทียบ

#### 🔴 Grad-CAM (Gradient-weighted Class Activation Mapping)
* **ข้อดี:** เป็นเทคนิคมาตรฐานที่ใช้งานง่าย คำนวณความสำคัญโดยอ้างอิงจากเกรเดียนต์ของคลาสเป้าหมายที่ไหลย้อนกลับเข้ามาที่เลเยอร์เป้าหมาย (Class-discriminative)
* **จุดด้อย/ข้อจำกัด (Weakness):** 
  * เมื่อมีวัตถุประเภทเดียวกันหลายชิ้นในภาพ (เช่น สุนัขหลายตัว) แผนภาพความร้อนมักจะไฮไลต์เฉพาะวัตถุตัวใดตัวหนึ่งที่เด่นที่สุด หรือไฮไลต์ได้ไม่ครบถ้วน เนื่องจากใช้วิธี **Global Average Pooling** ในการหาค่าเฉลี่ยเกรเดียนต์ ซึ่งเจือจางสัญญาณความร้อน
  * ความร้อนมักจะออกมาเป็นปื้นเบลอๆ วงใหญ่ ไม่คมชัดตามขอบวัตถุ

#### 🟢 Grad-CAM++ (ปรับปรุงการจับหลายวัตถุ)
* **ข้อดี (Strength):** 
  * ถูกออกแบบมาเพื่อแก้ข้อจำกัดของ Grad-CAM ในการวิเคราะห์ภาพที่มี **"วัตถุชนิดเดียวกันหลายชิ้น (Multiple Objects)"** โดยคำนวณร่วมกับเกรเดียนต์อันดับสอง (Higher-order gradients) ทำให้น้ำหนักมีการกระจายตัวตามพิกเซลได้ดีขึ้น ส่งผลให้แผนภาพความร้อนสว่างขึ้นครอบคลุมวัตถุได้ครบทุกตัวในภาพ
* **จุดด้อย:** ยังมีความกว้างของความร้อนที่เป็นปื้นเบลอระดับหนึ่ง (ดีกว่า Grad-CAM ปกติเล็กน้อยแต่ยังเบลอกว่า Layer-CAM)

#### 🔵 Eigen-CAM (เสถียร รวดเร็ว ไม่ใช้เกรเดียนต์)
* **ข้อดี (Strength):** 
  * **Gradient-free:** ไม่ใช้เกรเดียนต์เลย แต่ใช้หลักการหาองค์ประกอบหลักที่ 1 (**PCA**) บนฟีเจอร์แมปโดยตรง ทำงานได้เร็วมาก และแก้ปัญหาเรื่องเกรเดียนต์หายหรือการรบกวนของสัญญาณเกรเดียนต์ได้ 100%
  * สามารถส่องรูปร่างและขอบเขตโดยรวมของวัตถุเด่นทั้งหมดได้อย่างแม่นยำและเสถียร
* **จุดด้อย/ข้อจำกัด (Weakness):** 
  * **Non-class discriminative:** เนื่องจากไม่ได้ใช้เกรเดียนต์ของคลาสที่เจาะจง มันจึงไฮไลต์วัตถุทุกอย่างที่เด่นขึ้นมาพร้อมกันหมดโดยไม่แยกแยะว่าคลาสไหนคือสุนัขหรือแมว (เช่น แม้ว่าเราจะสั่งให้ตรวจเฉพาะสุนัข แต่มันก็จะส่องความร้อนบนตัวแมวขึ้นมาด้วย)

#### 🟡 Layer-CAM (ยอดฝีมือความคมชัดระดับพิกเซล)
* **ข้อดี (Strength):** 
  * **Fine-grained details:** ใช้น้ำหนักเกรเดียนต์รายพิกเซลโดยตรง (Spatial-wise) ทำให้แผนภาพความร้อนมีความละเอียดสูง คมชัด เจาะจงตามพื้นที่ของวัตถุจริง ไม่เป็นปื้นเบลอ
  * ทำงานได้ดีกับทุกเลเยอร์ของโมเดล (ทั้งเลเยอร์ระดับตื้นและลึก)
  * สามารถจับวัตถุประเภทเดียวกันหลายชิ้นได้คมชัดและสมบูรณ์แบบโดยยังรักษาสิทธิ์ความจำเพาะเจาะจงของคลาสเป้าหมายไว้ได้
* **จุดด้อย:** เนื่องจากให้ดีเทลละเอียด บางครั้งความร้อนอาจแยกย่อยเป็นจุดเล็กๆ หลายจุดแทนที่จะเชื่อมโยงกันเป็นวัตถุชิ้นเดียวหากตั้งค่าเลเยอร์ไม่เหมาะสม

### ⚖️ การเปรียบเทียบระหว่าง YOLOv8-nano และ YOLO11-nano (SIDA Project Context)

| คุณสมบัติ | YOLOv8-nano (`yolov8n`) | YOLO11-nano (`yolo11n`) |
| :--- | :--- | :--- |
| **สถาปัตยกรรม (Backbone/Neck)** | ใช้บล็อก **C2f** (Cross Stage Partial Bottleneck) | ใช้บล็อก **C3k2** (เร็วกว่าและมีประสิทธิภาพโครงสร้างดีขึ้น) |
| **กลไกความใส่ใจ (Attention)** | ไม่มีกลไก Attention ในระดับโครงสร้างโมเดลพื้นฐาน | มีการผสานเลเยอร์ **C2PSA** (Channel-to-Pixel Spatial Attention) ซึ่งใช้กลไก Multi-head Self-Attention เพื่อวิเคราะห์ความสัมพันธ์ระยะไกลของฟิกเซล |
| **จำนวนพารามิเตอร์** | ~3.2 ล้านพารามิเตอร์ | ~2.6 ล้านพารามิเตอร์ (มีขนาดโมเดลเล็กลงแต่ฉลาดขึ้น) |
| **ความแม่นยำ (COCO mAP)** | ปานกลาง (มาตรฐาน YOLOv8) | สูงกว่า YOLOv8-nano ประมาณ 1-3% mAP |
| **พฤติกรรมในแผนภาพความร้อน (CAM)** | แผนภาพความร้อนจะเกาะกลุ่มกันตามฟีเจอร์คอนโวลูชันปกติ | ด้วยกลไกของ C2PSA แผนภาพความร้อนสามารถวิเคราะห์พิกเซลที่มีผลสำคัญในระยะไกลและขอบเขตวัตถุได้แม่นยำขึ้น |

2. ย้ายไปยังโฟลเดอร์โปรเจกต์สัปดาห์ที่ 1:
   ```powershell
   cd c:\Users\Acer\Desktop\Project\Week1
   ```
3. รันสคริปต์เตรียมระบบ:
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process
   ```

## 🚀 ขั้นตอนการทดลองรันสคริปต์ (Running the Experiment)

1. **เปิดใช้งานสภาพแวดล้อมจำลอง (Activate Environment):**
   * หากตั้งค่าสำเร็จด้วย **Conda**:
     ```powershell
     conda activate yolo_gradcam
     ```
   * หากตั้งค่าด้วย **Virtualenv** (`.venv`):
     ```powershell
     .\.venv\Scripts\Activate.ps1
     ```
2. **รันการทดลอง:**
   ```powershell
   python gradcam_yolo.py
   ```
3. **ตรวจสอบผลลัพธ์:**
    * สคริปต์จะทำการตรวจสอบรูปภาพตัวอย่าง `sample_cat_dog.jpg` (หากไม่มีจะดาวน์โหลดให้ทันที)

    * คำนวณแผนภาพความร้อน 4 รูปแบบ (GradCAM, GradCAMPlusPlus, EigenCAM, LayerCAM) สำหรับแต่ละโมเดล
    * รูปภาพผลลัพธ์ความร้อนทั้งหมดจะถูกเซฟลงในโฟลเดอร์ `outputs/` ในรูปแบบ:
      * `outputs/yolov8n_[method]_result_0.jpg`
      * `outputs/yolo11n_[method]_result_0.jpg`
