# รายงานการทดลอง Shape Reasoning v2

## วัตถุประสงค์

ทดสอบว่า YOLO11n สามารถตรวจจับวงกลม สามเหลี่ยม และสี่เหลี่ยมได้อย่างถูกต้อง
ในภาพที่มีวัตถุเดี่ยวและหลายวัตถุหรือไม่ จากนั้นทดสอบว่า Grad-CAM และการทดลอง
แบบ counterfactual สนับสนุนคำอธิบายเชิงเรขาคณิตของโมเดลหรือไม่

## สิ่งที่ปรับปรุงจาก baseline เดิม

โมเดลเดิมถูกฝึกด้วยภาพที่มีวัตถุเดี่ยว แต่ถูกทดสอบกับภาพที่มีวัตถุสามชิ้น
ดังนั้น Dataset v2 จึงเพิ่มภาพหลายวัตถุ ภาพที่มีรูปทรงชนิดเดิมซ้ำกัน กรณีที่ยาก
ภาพพื้นหลังเปล่า object mask, boundary mask, rationale mask และตัวตรวจสอบ
คุณภาพ dataset

ชุดภาพควบคุมภายนอกแบบ locked ถูกเก็บไว้ที่
`data/shapes_v2/locked_original/` ส่วนโครงสร้างข้อมูล v1 ที่ไม่ใช้งานแล้วถูกลบ
หลังจากย้ายชุดควบคุมเรียบร้อย

## การตรวจสอบคุณภาพ Dataset

Dataset v2 ประกอบด้วยภาพสำหรับฝึก 295 ภาพ ภาพ validation 48 ภาพ ภาพทดสอบ
วัตถุเดี่ยว 30 ภาพ และภาพหลายวัตถุแบบ locked-style 30 ภาพ รวม annotation ของ
วัตถุทั้งหมด 558 รายการ มีภาพพื้นหลังสำหรับ train 10 ภาพ และสำหรับ validation
3 ภาพ

ตัวตรวจสอบรายงานว่าไม่พบ label ผิดรูปแบบ ไม่พบ bounding box ผิดพิกัด ไม่พบไฟล์
ภาพที่อ่านไม่ได้ และไม่พบภาพซ้ำข้าม split

## ผลการตรวจจับ

ตัวประเมินผลที่พัฒนาขึ้นใช้การจับคู่ prediction กับ ground truth แบบ one-to-one
ด้วย IoU และแยกกรณีจำแนกคลาสผิดออกจากกรณีตรวจจับวัตถุซ้ำ

| Dataset | Precision | Recall | นับจำนวนถูกต้องทั้งหมด | จำแนกคลาสผิด | ตรวจจับซ้ำ |
|---|---:|---:|---:|---:|---:|
| baseline เดิม / locked multi-test | 0.806 | 0.922 | 0.600 | 7 | 13 |
| baseline_v2 / v2 multi-test | 1.000 | 1.000 | 1.000 | 0 | 0 |
| baseline_v2 / original locked multi-test | 1.000 | 1.000 | 1.000 | 0 | 0 |
| baseline_v2 / robustness variants | 1.000 | 1.000 | 1.000 | 0 | 0 |

ความล้มเหลวหลักของโมเดลเดิมเกิดจากความแตกต่างระหว่างข้อมูลฝึกกับข้อมูลทดสอบ
(distribution shift) ไม่ใช่เกิดจากการตั้งค่า NMS เพียงอย่างเดียว การเพิ่มภาพ
หลายวัตถุเข้าไปในชุดฝึกช่วยกำจัดปัญหาการสับสนคลาสและการตรวจจับวัตถุซ้ำที่พบใน
ชุดทดสอบหลายวัตถุทั้งสองชุด

## การปรับค่า Confidence และ NMS

จากการทดลองกวาดค่าบน validation ได้ค่า confidence ที่ `0.20` และค่า NMS IoU
ที่ `0.50` ค่า configuration นี้ยังให้ Precision, Recall และความถูกต้องของจำนวน
วัตถุเท่ากับ 100% บน locked original multi-test และ robustness variants

การเลือกค่าดังกล่าวทำจาก validation ส่วน locked test ใช้สำหรับยืนยันผลเท่านั้น

## ผลการทดลอง Counterfactual เชิงเรขาคณิต

มีกรณี counterfactual ทั้งหมด 70 คู่ โดยคาดว่าความมั่นใจของคลาสเป้าหมายจะลดลง
เมื่อทำลายลักษณะทางเรขาคณิตที่ใช้จำแนกโดยเจตนา ขณะที่ control ที่ไม่ได้เปลี่ยน
ลักษณะเรขาคณิตควรมีความมั่นใจคงเดิม

| กรณีทดสอบ | อัตราผ่าน | ค่า confidence delta เฉลี่ย |
|---|---:|---:|
| วงกลม → รูปหลายเหลี่ยม 8 ด้าน | 80% | -0.227 |
| วงกลม → รูปหลายเหลี่ยม 6 ด้าน | 100% | -0.986 |
| สามเหลี่ยม → ตัดยอดมุมออก | 100% | -0.359 |
| สี่เหลี่ยม → ทำมุมให้โค้งมน | 100% | -0.992 |
| Control วงกลมที่ไม่เปลี่ยนแปลง | 100% | 0.000 |
| Control สามเหลี่ยมที่ไม่เปลี่ยนแปลง | 100% | 0.000 |
| Control สี่เหลี่ยมที่ไม่เปลี่ยนแปลง | 100% | 0.000 |
| รวมทั้งหมด | 97.14% | — |

ผลนี้สนับสนุนว่าโมเดลมีความไวต่อการเปลี่ยนแปลงทางเรขาคณิตที่ชัดเจน กรณีวงกลม
เปลี่ยนเป็นรูป 8 ด้านถูกออกแบบให้ยาก เพราะรูปแปดเหลี่ยมมีลักษณะใกล้เคียงวงกลม
จึงควรนำไปศึกษาเพิ่มเติม ไม่ควรสรุปว่าเป็นความล้มเหลวของโมเดลทั้งหมด

## ผลการทดลอง Grad-CAM

สร้าง Grad-CAM สำหรับวัตถุที่จับคู่กับ ground truth ได้ถูกต้องทั้งหมด 90 ชิ้น
จาก original locked multi-test โดยเก็บผลไว้ที่
`outputs/shapes_v2/baseline_v2/cam/gradcam/`

| คลาส | พลังงานภายในวัตถุ | พลังงานบนวัตถุอื่น | พลังงานบนบริเวณเหตุผล | Top-CAM IoU |
|---|---:|---:|---:|---:|
| วงกลม | 0.823 | 0.096 | 0.038 | 0.029 |
| สามเหลี่ยม | 0.386 | 0.250 | 0.005 | 0.009 |
| สี่เหลี่ยม | 0.610 | 0.241 | 0.003 | 0.008 |

ผลการทดสอบ faithfulness ด้วยการลบบริเวณที่ CAM ให้ความสำคัญ:

| คลาส | Confidence drop จาก Top-CAM | Confidence drop จากการสุ่ม |
|---|---:|---:|
| วงกลม | 0.267 | 0.009 |
| สามเหลี่ยม | 0.063 | -0.001 |
| สี่เหลี่ยม | 0.241 | 0.001 |

การลบบริเวณ Top-CAM ทำให้ confidence ลดลงมากกว่าการลบบริเวณแบบสุ่มในทุกคลาส
จึงสนับสนุนว่าบริเวณที่ CAM เลือกมีความเกี่ยวข้องเชิงสาเหตุกับการตรวจจับ
อย่างไรก็ตาม ผลนี้ยังไม่พิสูจน์ว่าโมเดลกำลังนับจำนวนมุมอย่างชัดเจน เพราะ CAM
ของสามเหลี่ยมและสี่เหลี่ยมยังซ้อนทับกับ rationale mask น้อย และยังมีพลังงาน
บางส่วนไปอยู่บนวัตถุอื่น

## สรุปตามเกณฑ์การประเมิน

### Detection Gate

ผ่าน โมเดลสามารถแยกรูปทรงทั้งสามชนิดได้ถูกต้องในเงื่อนไขภาพหลายวัตถุและ
robustness ที่ทดสอบ

### Reasoning Gate

ผ่านบางส่วน การทดสอบ counterfactual สนับสนุนว่าโมเดลไวต่อรูปทรงเรขาคณิต และ
การทดสอบ faithfulness สนับสนุนว่าบริเวณที่ CAM เลือกมีความเกี่ยวข้องกับผลลัพธ์
แต่ heatmap ยังไม่สามารถระบุมุมหรือเส้นขอบตามเหตุผลที่มนุษย์กำหนดได้อย่างชัดเจน

สำหรับวงกลม การไม่มีมุมไม่สามารถแทนด้วยบริเวณ positive เพียงจุดเดียวได้ จึงควร
ประเมินผ่านการทดลองเปลี่ยนเส้นขอบหรือ contour counterfactual แทนการคาดหวังให้
ทั้งวงกลมเป็นสีแดง

## ขั้นตอนทางวิทยาศาสตร์ถัดไป

ที่มี supervision ด้านเรขาคณิต หรือเพิ่มงาน segmentation/keypoint แล้วเปรียบเทียบ
